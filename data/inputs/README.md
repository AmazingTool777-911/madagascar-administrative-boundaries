# ADM Seeding Inputs

This directory contains the administrative (ADM) value inputs for Madagascar, structured by level (Provinces, Regions, Districts, Communes, and Fokontanys).

## Purpose

The data stored here serves two primary purposes:
1. **Database Seeding**: These files provide the raw administrative hierarchy and spatial data (geometries) required to seed the ADM tables in a relational database (e.g., PostgreSQL/PostGIS).
2. **Catalog Generation**: The inputs are used to generate public-facing catalog data, providing a structured reference for administrative boundaries and metadata.

## Provenance

The contents of this directory are automatically generated by the extraction pipeline:
- **Source Script**: [extract-adm-inputs.script.ts](file:///home/tolotra/it/projects/mada-adm/scripts/extract-adm-inputs/extract-adm-inputs.script.ts)
- **Source Data**: Typically extracted from official GeoJSON datasets or upstream administrative boundary sources.

## Data Format

The files are usually stored as JSON arrays of administrative values, specifically designed to be consumed by the project's DML (Data Manipulation Layer) adapters. Each record contains:
- Administrative name (e.g., `province`, `region`, etc.)
- Parent references (e.g., `region_id`, `province_id`)
- Spatial geometry (as raw GeoJSON objects)
- Administrative level markers

## Usage

To update the data in this directory, run the extraction script:
```bash
deno run -A scripts/extract-adm-inputs/extract-adm-inputs.script.ts [options]
```

> [!IMPORTANT]
> This directory is considered a "generated input" source. While it can be tracked in version control to provide a stable baseline for seeding, any manual edits should be carefully reviewed as they may be overwritten by subsequent script executions.
