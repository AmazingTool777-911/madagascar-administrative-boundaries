# ADM Seeding Inputs

This directory contains the administrative (ADM) value inputs for Madagascar,
structured by level (Provinces, Regions, Districts, Communes, and Fokontanys).

## Purpose

The data stored here serves two primary purposes:

1. **Database Seeding**: These files provide the raw administrative hierarchy
   and spatial data (geometries) required to seed the ADM tables in a relational
   database (e.g., PostgreSQL/PostGIS).
2. **Catalog Generation**: The inputs are used to generate public-facing catalog
   data, providing a structured reference for administrative boundaries and
   metadata.

  upstream administrative boundary sources.

## Data Format

The files are stored as Newline Delimited JSON (NDJSON), where each line is a
self-contained administrative record. The format is strictly
**database-agnostic**: it does not contain structural foreign keys or internal
database IDs. Instead, parent-child relationships (e.g., which District belongs
to which Region) are determined solely by the descriptive attributes (names) of
the features.

Each record contains:

- Administrative names (e.g., `fokontany`, `commune`, `district`, `region`,
  `province`)
- Spatial geometry (as raw GeoJSON objects)

## Usage

To update the data in this directory, run the extraction script:

```bash
deno run -A scripts/extract-adm-inputs/extract-adm-inputs.script.ts [options]
```

> [!IMPORTANT]
> This directory is considered a "generated input" source. While it can be
> tracked in version control to provide a stable baseline for seeding, any
> manual edits should be carefully reviewed as they may be overwritten by
> subsequent script executions.
